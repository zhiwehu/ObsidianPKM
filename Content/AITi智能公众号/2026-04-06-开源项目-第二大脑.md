# 我开源了一个"第二大脑"项目，搭配 OpenClaw 使用效果翻倍

**太长不看版：**

我把 Karpathy 的 LLM Wiki 方法论 + PARA 知识分类法整合成了一个开源项目，配合 OpenClaw 或 Claude Code 使用，实现了"把什么都丢给 AI，AI 自动整理分类"的效果。

GitHub：github.com/zhiwehu/second-brain

---

## 痛点

你是不是也有这些困扰？

- 收藏夹里堆了 500+ 篇文章，再也没打开过
- 突然想起"我之前看过那个观点"，但怎么都想不起来在哪
- 笔记软件文件夹越建越多，自己都乱了
- 想整理但每次都拖延，因为太麻烦了

**我之前也是这样。**

---

## 解决方案

我把 Karpathy（前 OpenAI、 Tesla AI 负责人）的 **LLM Wiki** 方法论，和 Tiago Forte 的 **PARA** 知识分类法，整合成了一个开源的"第二大脑"系统。

简单说就是：**把推文、文章、图片、语音、会议记录、想法——全都丢给 AI，AI 自动帮你整理、分类、建立链接。**

**使用方式：**
只需要对 OpenClaw 说一句话：
- "帮我把这条推文存到第二大脑"
- "把今天的会议记录存进去"
- "把那张截图保存"

**AI 自动完成：**
1. 提取内容、生成摘要
2. 按 PARA 分类（项目/领域/资源/归档）
3. 写入知识库
4. 与已有内容建立链接
5. 更新索引和日志

---

## 核心原理

**1. LLM Wiki（Karpathy）**

不是一次性 RAG 检索，而是像维基百科一样**持续累积**：
- 每条新信息都被 AI 提炼要点
- AI 记住内容之间的关系，自动建立链接
- 知识越累积越丰富，而不是越存越乱

**2. PARA 方法（Tiago Forte）**

把你的信息分成四类：

| 分类 | 含义 | 例子 |
|------|------|------|
| Projects | 正在做的项目 | 产品开发、学习计划 |
| Areas | 持续关注的领域 | 健康、财务、职业发展 |
| Resources | 感兴趣但暂不行动 | 研究主题、待读文章 |
| Archives | 已完成或放弃的 | 旧项目、历史资料 |

**AI 帮你决定每条信息该去哪个类，不用自己想。**

## 技术细节

**架构：**
- 三层 Memory 架构（Token 节省，不需要每次把全部内容给 AI）
- 支持 7 种内容类型：推文、文章、图片、语音、文件、聊天记录、日程
- 所有数据本地存储，不上传云端

**可选工具：**
- Whisper / SenseVoice：语音转文字
- yt-dlp：YouTube/B站字幕
- agent-reach：Twitter/小红书/公众号内容获取
- exiftool：图片元数据
